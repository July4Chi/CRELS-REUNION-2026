## 1. 研究ノートの標準4部構成

新規の研究ノートは、次の構成に従う（Rev 1.2 テンプレート）：

1. **事実レイヤー** — 独立に確認可能な事実のみを、出典付きで記述する。仮説と混在させない。
2. **仮説レイヤー** — 未検証仮説であることを明記した上で、逆算・解釈を記述する。
3. **対話原文／一次記録** — AI対話ログ等は日付を固定し、以後改変しない。
4. **Future Work** — 開放端・今後の検証課題を明示する。

## 2. ラベリング規則

- **【未検証仮説】** — 実験・査読による検証を経ていない主張には、必ずこのラベルを付す。「内部整合性がある」ことと「検証済みである」ことは区別する。
- **AI対話由来の評価値** — 「Verified by ○○」「99% Consistency」「Feasibility 85%」など、AIチャットの応答に由来する数値・評価は**検証ではない**。引用する場合は「AI対話ログの記述（未検証）」であることを明記する。
- **健康・生理効果に関わる主張**（例: 528Hz と DNA 修復）— 科学的に実証されていない旨を併記する。
- **実験プロトコル** — 有害物質（例: ソテツに含まれるサイカシン）を扱う手順には安全警告を必須とする。
- **考古学的解釈** — 学界の主流解釈と本プロジェクトの解釈仮説を区別して記述する（Rev 1.2 の事実レイヤーが模範例）。

## 3. 一次記録（Primary Source）の扱い

- 日付固定の対話ログ・記録文書は**原文を保存し、遡及編集しない**。
- 本基準の制定以前に書かれた文書には、冒頭に **Documentation Status 注記**を付し、原文はそのまま保存する。注記の標準文面：

> **📌 Documentation Status（追記日付）:** 本文書は編集基準（EDITORIAL_STANDARD.md）制定以前に書かれた一次記録であり、原文を保存している。文中の「Verified」「Consistency」「Feasibility」等の評価は AI 対話セッションの応答に由来し、実験・査読による検証を意味しない。本文書に含まれる物理学的主張は【未検証仮説】として読まれたい。

## 4. Status ヘッダの標準値

各文書のヘッダ `**Status:**` には、次のいずれかを用いる：

| Status | 意味 |
|---|---|
| 一次記録（Primary Source） | 日付固定・以後追記なしの記録 |
| 研究ノート | 事実／仮説を分離して記述する現行文書 |
| 設計仕様（未検証仮説） | `hardware/` 配下の設計文書 |
| デモコード | 概念のPython表現（物理的検証を伴わない） |

## 5. 本基準の狙い

本プロジェクトは「オープンな知の分散配布」を原則に掲げている。その信頼性は、**主張の強さではなく、事実と仮説を誠実に区別する編集姿勢**によって支えられる。否定的な実験結果も歓迎し、検証されるまで仮説は仮説として提示する。
